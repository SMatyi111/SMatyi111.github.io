Privacy policy

What the game needs

TraceSpark uses Pi authentication to confirm that the game runs in the Pi environment. Authentication requests no optional scopes. The game does not store or transmit your Pi UID, username, wallet address, access token, passphrase, payment information, contacts, or location.

Local game data

Your device stores daily completion, best score, streak, analytics choice, and—only if analytics is allowed—a random installation identifier. This supports the game and can be erased using Data settings.

Optional gameplay analytics

Analytics is off until you allow it. If allowed, the random installation identifier is hashed in your browser. We receive the hash, event name, UTC date, game mode, score or streak where relevant, and app version. This lets us count distinct completions and whether the same installation returns on another day. The hash is pseudonymous: it is not your Pi identity, but it can distinguish one installation over time.

Hosting infrastructure may process normal request metadata such as IP address and user agent for security and operations. TraceSpark does not add either field to its analytics event records. Runtime logs are used only for this validation experiment and should be deleted or aggregated after the experiment review.

Sharing

The Share result button opens your device's share sheet or copies result text. TraceSpark does not read your contacts or post without your action. A share event is recorded only when optional analytics is enabled and the share or copy action completes.

Your choices

You may play without analytics. Open Data settings on the game screen to change your choice or erase all local game data. Erasing local data prevents future events from linking to the old installation hash; existing aggregate or runtime records cannot be located by Pi account because no Pi identity is collected.

Contact and status

TraceSpark is an independent Testnet research prototype, not an official Pi Network product. Privacy questions may be filed through the project's public app support channel before ecosystem submission. This policy will be updated with a durable support address before public listing.
